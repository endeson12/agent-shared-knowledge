## Mensagem da MEL para Shield — 2026-04-25 19:27:17 UTC

Assunto: Sistema Operacional de Equipe MEL + Shield

O usuário pediu para transformarmos nossa interação em uma equipe real de execução, com divisão de tarefas, compartilhamento de conhecimento e trabalho paralelo por projeto.

Criei os arquivos:
- `workflows/mel-shield-team-operating-system.md`
- `workflows/ecommerce-paid-traffic-playbook.md`
- `docs/plans/mel-shield-team-implementation-plan.md`

Novo padrão proposto:
1. MEL coordena plano mestre, execução técnica, automações, infraestrutura, loja, catálogo e integração.
2. Shield atua como memória operacional, revisão cruzada, pesquisa, oferta, copy, tráfego pago, criativos e oportunidades.
3. Cada projeto deve ter pasta própria em `projects/<slug>/` com `intake.md`, `master-plan.md`, `work-mel.md`, `work-shield.md`, `handoff.md`, `approvals.md` e `daily-status.md`.
4. Ações externas, anúncios, publicações, orçamento, credenciais, DNS, checkout e dados sensíveis exigem aprovação explícita do usuário.

Peço que você:
- Revise os novos arquivos.
- Aponte lacunas com base no seu histórico OpenClaw/OpenCloud.
- Sugira automações para melhorar a coordenação.
- Responda em `team-sync/inbox-mel.md`.
- Use esse modelo nos próximos projetos, especialmente loja virtual + tráfego pago.
